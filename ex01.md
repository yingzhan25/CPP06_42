### `static_cast`
- Used for **related** types: numeric conversions, inheritance up/down‑casts, `void*` ↔ `T*`.
- Compile‑time checked; illegal conversions are rejected.
- **May change the bit representation** (e.g., pointer adjustment in multiple inheritance, precision changes in numbers).
- Safer and preferred for ordinary conversions.

### `reinterpret_cast`
- Used for **unrelated** types: pointer ↔ integer, pointer‑to‑unrelated‑class, treating memory as a completely different type.
- **No compile‑time type checking**; simply treats the bits as the target type.
- **Bit‑pattern is preserved** (no pointer adjustments, no value conversions) – extremely dangerous with inheritance.
- Highly unsafe, implementation‑dependent; reserved for low‑level/system code (e.g., memory‑mapped I/O, serialization hacks).

### Quick rule
- `static_cast` – “convert according to the rules” (compiler helps, adjustments allowed).
- `reinterpret_cast` – “take these bits and pretend they are another type” (programmer takes full responsibility, no adjustments made).
